数据安全管理制度 v2.0
文档编号：SEC-2026-002
生效日期：2026-01-15

一、数据分级
- L1公开级：产品文档、公开API文档
- L2内部级：技术方案、项目报告、代码仓库（默认）
- L3机密级：用户隐私数据、财务数据、模型参数
- L4绝密级：加密密钥、生产环境凭证

二、访问控制
1. 最小权限原则：默认无权限，按需申请
2. 所有内部系统必须通过 SSO + MFA 认证
3. L3及以上数据访问需审批（直属上级+安全负责人）
4. API Key 统一由 Vault 管理，禁止硬编码或提交 Git

三、数据存储
- 用户数据：国内用户数据存储在中国区 AWS RDS，加密存储（AES-256）
- 日志数据：保留90天，之后自动归档到 S3 Glacier
- 备份策略：每日全量备份+每小时增量备份，异地容灾
- 数据库密码、API Key 等凭证必须通过机密管理服务获取

四、安全审查
- 每季度进行渗透测试（外部安全公司）
- 每半年进行合规审计（ISO 27001）
- 生产环境变更必须通过变更审批
- 离职员工权限必须在最后工作日当天回收
