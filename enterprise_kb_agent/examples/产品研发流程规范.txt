产品研发流程规范 v4.2
文档编号：PROC-2026-007
生效日期：2026-03-01

一、研发阶段划分
1. 需求阶段（1-3天）：产品输出PRD，技术评审，产出技术方案
2. 设计阶段（2-5天）：UI设计 + 技术详设（DB schema、API 契约、架构图）
3. 开发阶段（5-15天）：编码 + 单元测试（覆盖率≥70%）+ 自测报告
4. 测试阶段（3-7天）：QA功能测试、性能压测、安全扫描
5. 上线阶段（1天）：灰度10%→观察30分钟→全量，必须有回滚方案

二、技术规范
- 后端: Java 17+Spring Boot 3 / Python 3.11+FastAPI
- 前端: React 18+TypeScript+Ant Design 5
- 数据库: PostgreSQL 15，禁止存储过程，复杂逻辑放应用层
- 缓存: Redis Cluster，热点数据TTL≤5分钟
- 消息队列: RocketMQ，用于异步解耦（订单通知、数据同步）

三、质量门禁
- SonarQube 代码扫描无阻断性问题
- 核心接口 P99 延迟<500ms
- 单测覆盖率≥70%，核心模块≥85%
- Code Review 至少1人approve
